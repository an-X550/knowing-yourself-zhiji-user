# 知己

> 写下真实经历，看见反复出现的模式，尝试一个小行动，再用后续结果判断它是否真的适合你。

知己是一套面向中文个人日志的 AI 复盘工具。它可以帮你连接散落在不同日期里的线索，把“我最近好像不太对”变成更具体的观察和行动。

它不只是总结日志，也不会声称比你更了解你。它的工作方式是：

```text
记录 → 发现模式 → 形成行动 → 后续验证 → 沉淀认识
```

真正有价值的结果不是一份写得漂亮的报告，而是一个你原本没有看见的模式，促成了一次可以观察结果的改变。

## 开始前需要什么

你需要一个能够打开项目目录、读取项目指令并与 AI 对话的工具，例如 Claude Code、Codex、Trae 或 WorkBuddy。

开始前：

1. 下载或复制完整的知己用户版目录。
2. 用你的 AI 工具打开这个目录，而不是只打开其中一个文件。
3. 确认工具允许读取和写入项目内的 Markdown 文件。
4. 准备一篇真实日志；没有日志也可以使用 [`examples/demo/sample-journal.md`](examples/demo/sample-journal.md)。

不需要安装数据库，也不需要先学习所有命令。大多数时候，直接用中文描述需求即可。

## 第一次使用：六步完成最短闭环

1. **打开目录**：让 AI 在知己用户版目录中工作。
2. **写下或粘贴日志**：模板和自由叙事都可以。
3. **请求分析**：输入 `分析这篇日志`。
4. **读一个关键洞察**：先判断它是否有日志证据，而不是看它是否“说得深刻”。
5. **选择一个小实验**：采纳、修改或拒绝 AI 的建议都可以。
6. **记录结果**：在后续日志中写明做没做、发生了什么、是否出现反例。

你可以直接复制下面这段：

```text
日志：
今天本来想完成方案，但我一直在整理资料，直到晚上也没开始正文。

请帮我：
1. 找出一个我可能没看到的模式；
2. 说明哪些是事实，哪些只是推断；
3. 给我一个明天可以验证的小实验。
```

如果你已经把日志放进 `日志/`，也可以说：

```text
请帮我做今天的日志反馈，并检查上一次建议有没有得到验证。
```

## 日志怎么写

知己不要求固定模板。你可以写完整复盘，也可以只写几句话。自由叙事建议用以下任一前缀开始，以便稳定进入日志存档和日反馈流程：

```text
日志：……
日记：……
记录一下：……
```

有助于分析的信息包括：

- 发生了什么；
- 你当时怎么想、怎么感受；
- 你实际做了什么；
- 结果和预期有什么差异；
- 你还不确定什么。

这些不是填写任务。状态不好时，写三句话也足够：发生了什么、现在是什么感受、明天想尝试什么。

如果证据较少，知己会降低结论强度，明确提示“证据不足”，并给出更小的观察建议；它不应该为了显得聪明而补完你的故事。

## 你可以使用的功能

### 统一入口：不知道该从哪里开始

适合：你有一篇或一批材料，但不知道应该做日、周、月、项目、年度复盘，还是方向校准。

你可以说：

```text
请判断这些材料现在最适合做哪种复盘。
```

对应命令：`/review`

### 单日反馈：完成最快的行动闭环

适合：刚写完今天的日志，希望看见一个关键模式，并为明天设计一个小实验。

主要得到：昨日行动验证、今日关键洞察、一个可执行动作和下次观察点。

你可以说：

```text
请帮我做今天的日志反馈。
```

对应命令：`/daily-review today`

### 日志教练：改善记录质量

适合：你想知道最近几天的日志是否包含足够事实、感受、原因、行动和结果，而不是继续增加分析报告。

主要得到：分析就绪度、六步法写作习惯、重复缺口和最值得改进的一件事。

你可以说：

```text
请看看我最近几天的日志质量，告诉我只需要优先改进什么。
```

对应命令：`/journal-coach`

### 周复盘：看见短期趋势

适合：已经积累几天日志，希望检查本周目标、结果、正负原因、重来方案和下一步。

主要得到：基于“复盘六问”的周度综合，以及本周行为假说的支持证据、反例和待验证项。

你可以说：

```text
请基于最近一周的日志做周复盘。
```

对应命令：`/weekly-review`

### 月复盘：识别更稳定的模式

适合：已经积累一个月左右的日志，希望跨周观察精力、压力、关系、学习、工作和行动验证。

主要得到：主题归并、关键转折、稳定模式、反例、验证沉淀和下月重点。

你可以说：

```text
请基于这个月的日志做月复盘。
```

对应命令：`/monthly-review`

如果需要控制分析深度，可以使用 `fast`、`standard` 或 `full` 模式。

### 项目复盘：总结一个阶段

适合：完成项目、版本、比赛、课程或其他明确阶段后，想判断目标是否实现以及哪里可以重来。

主要得到：目标回顾、结果评估、正负原因、重来演练和后续规划。

你可以说：

```text
请帮我复盘这个项目阶段，并区分证据和事后解释。
```

对应命令：`/project-review`

### 年度回顾：连接长期成长轨迹

适合：已经有较完整的月报，希望识别全年转折、优势、持续挑战和下一年方向。

主要得到：逐月回顾、关键转折、成长轨迹、日志与复盘能力趋势，以及新年重点。

你可以说：

```text
请基于今年的月报生成年度成长回顾。
```

对应命令：`/yearly-review`

年度回顾依赖足够的月度材料。材料不足时，先完成最近一次周报或月报通常更有价值。

### 人生设计：校准长期方向

适合：问题不是“今天效率不高”，而是长期目标、生活结构、精力来源或价值选择出现持续冲突。

主要得到：当前处境、真正的问题、方向指南针、能量地图、多个可能的人生原型、7 天实验和验证方式。

你可以说：

```text
我想检查最近的长期方向，请先基于已有日志找证据，再设计一个 7 天实验。
```

对应命令：`/life-design --quick`

### 主题思考：让观点持续演化

适合：你想与 AI 深入讨论任何长期问题，并希望以后继续沿用、检验或修正这次形成的认识。

你可以说：

```text
我想和你讨论：持续摄入信息正在怎样影响我的生活。先不要保存；讨论形成明确认识后，再问我是否写入主题思考库。
```

AI 会先帮助你区分事实、推断、依据、反例和未决问题。只有你确认归纳后，它才会在 `关于我/思考/` 动态创建或更新主题文件；主题不限于示例，也没有固定清单。以后遇到明显相关的问题时，AI 最多读取两个相关主题，并明确说明参考了哪些历史思考。你当前的表达始终优先于旧观点。

## 行为假说和干预实验有什么区别

知己不会把一次日志分析当成最终结论。它会区分五件事：

```text
观察：日志里实际发生了什么。
行为假说：多个观察可能共同说明什么。
干预实验：为了检查或改变模式，这一次具体做什么。
实验结果：这次做了没有，发生了什么。
当前结论：现有证据更支持什么，还缺什么。
```

例如：

```text
观察：近三次拖延都发生在任务边界模糊时。
行为假说：任务边界不清会提高启动阻力。
干预实验：明天只写出第一步，并执行 10 分钟。
实验结果：依然没有开始。
当前结论：这次方法没有奏效；继续观察精力、环境或任务价值。
```

这里不能直接写“行为假说已被证伪”。一次实验失败只说明这次方法没有奏效。行为假说需要跨时间证据，之后可能处于：

- **待验证**：目前只是合理解释；
- **有支持证据**：不同日期出现相似模式；
- **出现反例**：有些情境不符合，需要收窄假说；
- **已证伪**：积累的证据持续与假说矛盾。

## AI 做什么，你做什么

### AI 负责

- 从日志和历史反馈中提取事实与跨时间模式；
- 明确区分事实、推断、建议和证据不足；
- 把模式写成可以继续检查的行为假说；
- 提出低成本、可执行、能观察结果的干预实验；
- 在后续反馈和周期复盘中追踪支持证据与反例；
- 信息不足时降低结论强度，而不是补完故事。

### 你负责

- 提供尽量真实的记录，不为迎合 AI 改写自己的体验；
- 判断建议是否适合你，并决定采纳、修改或拒绝；
- 如实记录做了、没做或做了一部分；
- 在后续日志中写下结果、感受和反例；
- 保留对自身经历、价值和人生方向的最终解释权。

### AI 不负责

- 诊断身体或心理问题；
- 根据单篇日志宣布你的性格、动机或人生真相；
- 替你做医疗、心理、法律、财务或重大人生决定；
- 在缺乏行动与验证时，用更多报告制造“已经改变”的错觉。

遇到高风险或专业问题，请把知己当作整理材料和准备问题的工具，并向合格专业人士求助。

## 自然语言与命令对照

| 你的目的 | 可以直接这样说 | 快捷命令 |
|----------|----------------|----------|
| 不知道从哪里开始 | 帮我判断现在最适合做什么复盘 | `/review` |
| 分析今天 | 帮我做今天的日志反馈 | `/daily-review today` |
| 改进日志写法 | 看看最近几天的日志质量 | `/journal-coach` |
| 回顾一周 | 基于最近一周日志做周复盘 | `/weekly-review` |
| 回顾一月 | 基于这个月日志做月复盘 | `/monthly-review` |
| 回顾一个项目 | 帮我复盘这个项目阶段 | `/project-review` |
| 回顾全年 | 基于今年月报做年度回顾 | `/yearly-review` |
| 校准长期方向 | 我想重新检查最近的生活方向 | `/life-design --quick` |

命令只是快捷方式。只要自然语言已经说清楚需求，就不必记命令。

## 结果保存在哪里

| 目录 | 内容 |
|------|------|
| `日志/` | 你的原始日志 |
| `复盘/每日反馈/` | 单日反馈和日志教练报告 |
| `复盘/每周复盘/` | 周复盘 |
| `复盘/每月复盘/` | 月复盘及相关分析 |
| `复盘/项目复盘/` | 项目或阶段复盘 |
| `复盘/年度回顾/` | 年度成长回顾 |
| `复盘/人生设计/` | 方向校准结果 |
| `关于我/` | 长期上下文、目标和验证沉淀 |
| `关于我/思考/` | 用户确认后动态创建的主题认识、依据、反例、未决问题与演化记录 |

这些文件让下一次分析可以消费以前的证据，不必每次从零开始。

## 隐私保护

真实日志和大部分复盘输出默认不会进入 Git。`关于我/*.md` 中有少量空白种子文件随项目提供，方便首次运行；因为它们已经被跟踪，`.gitignore` 不能自动保护填写后的内容。

如果你会在这个仓库中填写真实画像，首次填写前运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/protect-private-data.ps1
```

它会在当前克隆中把相关种子文件标记为 `skip-worktree`，减少内容误入 Git diff 的风险。换电脑或重新克隆后需要再次执行。

这只是本地防误提交措施，不是加密、备份或访问控制。请不要把不愿交给当前 AI 服务商处理的敏感信息写入日志；重要资料应另行备份。

## 常见问题

### AI 说得很有道理，我应该直接相信吗？

不要。先检查它引用了哪些日志事实，再找反例，并用后续行动验证。验证前，它只是一个可能有用的假说。

### 我没有按建议行动，是不是失败了？

不是。“没做”本身也是信息。可能是行动门槛太高、时机不对，也可能是问题根因判断错了。把它如实写进下一篇日志，比假装完成更有价值。

### 日志太短，还能分析吗？

可以，但结论应更保守。补充“发生了什么、我怎么感受、我做了什么、结果怎样”中的一两项，通常就能明显提高分析质量。

### 找不到日志或结果文件怎么办？

确认 AI 打开的是整个知己目录，并告诉它日志日期或文件名。你也可以直接把正文粘贴到对话中，再要求保存和分析。

### 一定要使用命令吗？

不用。自然语言是推荐入口。命令适合高频使用或你希望明确指定分析类型时。

### 应该先做日反馈还是月报？

优先日反馈。没有持续的行动和验证记录时，生成更长的报告通常不会带来更多真实改变。

## 现在就开始

打开 [`examples/demo/sample-journal.md`](examples/demo/sample-journal.md)，复制其中任意一天，然后输入：

```text
分析这篇日志。请只给我一个最关键的洞察、一个可验证的小实验，并说明下次应该记录什么。
```

下一次写日志时，再补上一句：

```text
上次实验：[做了 / 没做 / 做了一部分]
结果：[发生了什么]
判断：[支持原假说 / 出现反例 / 仍待验证]
```
