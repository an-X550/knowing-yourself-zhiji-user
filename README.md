# 知己用户版

> 把今天的日志贴进来，我来帮你看你自己没发现的模式。

这是 **知己** 的用户试用版。它只保留使用所需文件，不包含开发记录、版本治理和内部规划文档。

## 适合做什么

- 分析当天日志，获得一条更清晰的洞察和一个可尝试的动作。
- 汇总一周或一个月的日志，生成复盘反馈。
- 对项目、长期方向或人生设计做阶段性复盘。

第一次使用时，建议先从当天日志开始。

## 快速开始

1. 将本仓库放到 Claude Code 可以读取的项目目录。
2. 打开 Claude Code，并进入本目录。
3. 直接粘贴今天的日志，或运行：

```bash
/review
```

如果你明确只想做当天反馈，也可以运行：

```bash
/daily-review
```

没有现成日志时，可先用 [`examples/demo/sample-journal.md`](examples/demo/sample-journal.md) 试跑。

## 常用入口

| 入口 | 什么时候用 |
|------|------------|
| `/review` | 不确定该选哪个时，优先用这个 |
| `/daily-review` | 分析当天日志 |
| `/weekly-review` | 做周复盘 |
| `/monthly-review` | 做月复盘 |
| `/project-review` | 做项目或版本复盘 |
| `/life-design` | 做长期方向和人生设计校准 |
| `/journal-coach` | 看最近几天的连续状态 |

## 反馈什么最有用

内测时不需要填表。你自然告诉维护者这些就够了：

- 哪里分析得准。
- 哪里有用，或者哪里没用。
- 哪一步让你不知道怎么继续。
- 后续日志里有没有回应、执行、修正或反驳前一次建议。
