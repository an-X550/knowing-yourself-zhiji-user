# 第一性原理复核契约

## 显式触发

仅当用户明确要求“依据第一性原理分析”、要求按第一性原理复核/压缩既有结论，或表达等价意图时启用。本契约覆盖默认表达深度，不覆盖证据、隐私、写入确认、安全边界和当前任务的稳定输出结构。

## 复核顺序

1. 指出待解决的真实问题，不复述现有方案或表面现象。
2. 区分已知事实、未验证假设、不可突破约束和价值取舍。
3. 检查是否沿用惯例、把现象当根因，或遗漏更小的替代路径。
4. 给出更直接的结论、替代解释或最小验证；证据不足时明确降级判断。

## 场景适配

- 日反馈：压缩到“事实 / 关键假设 / 最小验证”；不增加第二个洞察或行动，继续遵守 D0-D6、一个动作和长度上限。
- 周/月/项目复盘：保留复盘六问；将复核结果放入聊天摘要或对应章节，不新增一级标题。
- 主题思考与个人价值选择：分开价值、事实、约束和不可消除的代价；AI 只能指出冲突、反例、风险或证据缺口，不输出伪客观的唯一答案。

## 长度与表达

内容过长时，优先删除不改变判断、行动或验证的信息；不得为了展示推理过程复制完整步骤或重复既有结论。不得以礼貌性肯定替代判断；提出问题时同时说明证据、影响和替代路径。
