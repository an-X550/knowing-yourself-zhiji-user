# 日志输入契约

## 目标

让模板日志和自由叙事日志进入同一条日反馈链路。输入质量按可用证据判断，不按栏目、篇幅、文采或是否使用六步法模板判断。

## 保证触发的输入

满足任一条件即可视为明确日志意图：

1. 出现现有模板字段：`幸福日志`、`开心的事情`、`充实的事情`、`感谢的人`、`思考`、`待改进`、`ToDolist`。
2. 使用明确前缀：`日志：`、`日记：`、`记录一下：`。
3. 明确要求分析：`分析这篇日志`、`分析今天的日记`、`帮我看看这篇记录` 等。

不要用单独的“今天”“我”“记录”“发生了”等高频词自动触发。长自由文本没有明确日志意图时，只问一次“这是想作为日志存档并分析吗？”，不要直接写入文件。

## 日期识别

至少识别以下格式：

- 中文标题：`# 7月8日`、`# 7 月 8 日`
- ISO 标题：`## 2026-07-08`
- 斜杠日期：`2026/7/8`
- 日志前缀：`幸福日志 7.8`、`日志 7.8`
- 相对日期：`今天`、`昨天`

处理规则：

1. 两个及以上日期头走多日模式；一个日期头走单日模式。
2. “今天 / 昨天”必须换算为实际日期，并在内部记录日期来源为 `relative`。
3. 完全没有可靠日期时，只问一次日期；不得静默默认今天。
4. 日期只影响存档位置，不改变用户原文。

## 内部证据卡

读取目标条目后，在上下文中临时提取：

```text
date: 日期及来源（explicit / relative / confirmed）
facts: 用户明确写出的事件、行为和结果
states: 用户明确写出的感受、能量或身体状态
interpretations: 用户自己的解释、判断或疑问
intentions: 用户想保持、停止或尝试的行为
previous_action_evidence: 对上一条行动的明确执行 / 未执行 / 不确定证据
gaps: 会限制本次结论的信息缺口
```

不要创建证据卡文件。代理推断不得写入 `facts`、`states` 或 `previous_action_evidence`。

## A-D 输入证据等级

等级只控制分析深度，不展示给用户：

| 等级 | 最小证据 | 允许输出 |
|---|---|---|
| A 级 | 具体事实 + 明确状态 + 用户解释 + 行动意图或上一行动结果 | 完整日反馈；允许有证据的模式连接 |
| B 级 | 具体事实 + 状态或解释至少一项 | 一个核心洞察；跨日模式只能写成待验证假说 |
| C 级 | 只有简短事实、情绪或片段，但能确认是本人经历 | 镜像反射 + 一个维护闭环的小动作；不推断根因或长期模式 |
| D 级 | 无法确认日期、本人经历或具体事件 | 不生成正式日反馈；只问一个最能补齐证据的问题 |

自由叙事只要证据充分，同样可以是 A 级。固定栏目缺失不能作为降级理由。

## 降级输出规则

1. A 级：遵守完整日反馈契约。
2. B 级：保留一个核心洞察；没有跨日证据时删除 `🔗`。
3. C 级：只复述可核验的差距或状态，不写隐含动机，不连接长期模式。
4. D 级：返回一个补证问题，调用方不得保存空反馈或写回 `verified-patterns.md`。
5. 无论等级如何，缺少上一行动证据时只能判断 `⚠️ 证据不足`，不能把未记录等同于未执行。

## 存档边界

1. 保留原文，不把自由叙事重写成模板。
2. 单日追加前检查目标月文件是否已有同日条目；有重复风险时合并本次新内容或请用户确认，不盲目追加完整副本。
3. 多日拆分无法可靠归属的片段合并到最近一个已确认日期，并明确报告存在未识别片段。
