# 主题思考契约

## 适用边界

本契约只处理用户主动与 AI 探讨后形成、且经用户确认值得长期保留的认识。不从日志、日反馈或复盘自动摘录思考，不保存完整聊天记录，也不把 AI 单方面观点写成用户认识。

## 探讨与确认

讨论时区分事实、推断、建议和价值判断，检查依据、反例、适用边界与未决问题。AI 可以严肃挑战用户的价值判断和目标选择，但必须明确给出证据缺口、反例、推理跳跃、价值冲突、风险或替代解释之一；不能把 AI 偏好伪装成客观事实。形成可沉淀认识后，先向用户展示主题归纳与变更摘要。未经用户确认不得创建、写入或更新任何主题文件。

## 动态主题与文件安全

主题不使用白名单或预置分类，允许根据用户确认的主题名创建任意新主题。将主题名映射为 `context.thinking_topic` 前，去除 `/`、`\`、`:`、`*`、`?`、`"`、`<`、`>`、`|` 等路径分隔符或不安全字符，并确认解析后的目标仍位于 `context.thinking_dir` 内。

名称与已有主题相同或近义时，优先更新已有主题或请用户选择。不得越界写入，不得静默覆盖无关文件。首次保存时创建目录和 `context.thinking_index`；新主题同时写入主题文件与索引。

## 主题文件结构

主题文件包含 YAML frontmatter、一个前置执行入口，以及以下六个陈述式板块：

0. `0. 当前行动卡`：放在正文最前面，用表格或等价清单写清短期可执行动作、长期坚持动作、触发场景、停止线和下次复查条件；主题思考不默认定期回访或周反馈，读者不必读完整篇也能知道下次遇到同类问题时怎么做。
1. `1. 当前问题`：记录重复现象、情境、感受和冲突，不抢先解释原因；合并同义问题，但保留可回查的问题痕迹。
2. `2. 当前判断`：区分事实、推断和价值判断；AI 的观点不得伪装成用户认识。
3. `3. 依据来源与思考校正`：记录来源、反例、证据缺口、推理跳跃、价值冲突、风险或替代解释；严肃审查至少指出其中一项。
4. `4. 值得保留的行动`：记录有证据支持、符合用户价值或已有正向结果、值得长期反复使用的行动；必须使用“长期动作 / 解决的问题 / 具体做法 / 使用边界”四列，避免写成一次性任务、抽象口号或过细技巧。
5. `5. 下一次怎么做`：同时只保留一个进行中的行动实验，用编号步骤写明触发条件、最小动作、预期观察和验证时间；未经验证的方案不得写成答案。
6. `6. 我的观点演化路径`：新认识与旧观点冲突时，先展示“保留 / 修正 / 新增反例 / 仍未解决”的变更摘要；只有实质变化才记录观点演化，包含日期、旧观点、改变依据和新观点。

更新主题时优先修订前五部分。详细日志、文章和引用只保留来源入口，不把主题文件扩写成会话转录或资料仓库。

六个板块是帮助用户检查思考闭环的结构清单，不要求各部分篇幅相等。每一部分按证据、问题复杂度和行动需要决定具体程度：有明确行动或高风险判断时写细；证据不足、没有新认识或与当前行动无关时可以短写并说明原因。不得为了填满标题而平均扩写泛泛原则。主题文件必须使用 `0.` 到 `6.` 的编号标题固定阅读路径；`0. 当前行动卡` 应只做短期入口并引用 `4. 值得保留的行动` 中的长期动作，不重复堆叠完整说明；`4. 值得保留的行动` 和 `5. 下一次怎么做` 必须优先服务执行，写清具体对象、适用条件、做法、停止线或最小动作，让用户看完就能行动。

## 自动召回

普通提问涉及用户既有观点、长期困惑或价值判断时，先读取 `context.thinking_index`。只有主题、别名或核心问题明显相关时，才读取最多 2 个最相关主题文件；没有明显匹配就不读取详细主题。

回答必须明确告知用户参考了哪些历史主题。历史认识只是用户曾认可的起点，不是永久事实；当前表达与历史观点冲突时，以当前表达优先，指出可能的观点变化，并在需要更新时重新走确认流程。

召回历史主题时按“当前表达 -> 最近 90 天证据 -> 已验证长期模式 -> 90 天前未复核观点”的顺序确定权重。90 天前的具体观点不是自动失效，而是进入复查状态；引用时必须说明更新时间，并检查用户是否仍认可。核心价值、长期主题和已验证模式不因超过 90 天自动降级，但需要结合最近证据和反例判断是否仍适用。

## 高频链路边界

日反馈默认不读取思考库；只有当天日志明确涉及某主题，且不读取会导致重复或冲突建议时，才按需读取 1 个主题。周/月复盘只在思考已经影响行动、方向选择或反复困境时引用。

## 异常处理

- 索引存在但主题文件缺失：提示索引失效，不根据缺失内容回答。
- 多个近义主题：建议合并或请用户选择。
- 用户拒绝保存：不写文件，也不反复劝说。
- 用户请求删除：展示影响范围，确认后删除主题并更新索引。
- 主题长期未更新：说明更新时间并检查是否仍认可，不自动判定失效。
