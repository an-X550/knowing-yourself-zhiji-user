# 复盘综合契约

## 适用范围

本契约适用于周报、月报和项目复盘的综合层。综合层负责主题归并、交叉验证、方向校准和最终报告，不复制粘贴各视角段落。

## 稳定报告结构

综合报告必须保留：

- YAML frontmatter
- 标题
- `## 聊天摘要`
- 复盘六问显式一级标题
- `## 质量自检`

复盘六问一级标题：

1. `## 一、回顾目标`
2. `## 二、评估结果`
3. `## 三、分析原因（正向）`
4. `## 四、分析原因（负向）`
5. `## 五、重来演练`
6. 周报使用 `## 六、下周规划`，月报使用 `## 六、下月规划`，项目复盘使用对应后续规划标题。

## 六问写作协议

### 一、回顾目标

必须覆盖当前方向锚点、原始目标、实际追求目标（若有漂移）、目标合理性，以及与上周期追踪或假说的关系（如有）。

### 二、评估结果

必须覆盖客观结果、主观感受、他人反馈（若有）、达成度判断、方向锚点完成度 / 缺席项检查、验证沉淀结果。

### 三、分析原因（正向）

必须覆盖超预期部分、成功链条、被用户忽略或值得保留的优势，以及哪些做法值得放大。

### 四、分析原因（负向）

必须覆盖不符合预期部分、根因分析、自我评价偏差、证据边界 / 反例。

### 五、重来演练

必须覆盖如果重来一次会保留什么、会提前调整什么、最小替代动作是什么。`重来演练` 必须独立成节，不能并入规划。

### 六、后续规划

必须覆盖下周期目标、手段路径、检查方式 / 检查点、待验证假说、重要缺席项的硬约束或补记录要求。

## 方向锚点缺席检查

对每个方向锚点检查本周期是否有行动证据。缺席项本身不是失败，而是需要用户确认的校准信号。

状态只能使用以下五类：

- `有推进`：日志、日反馈或视角分析中有明确行动证据。
- `缺席-未执行`：方向重要，但缺少行动证据，且有拖延、替代投入或计划落空证据。
- `缺席-未记录`：用户提到做了，但日志缺少过程、数量或结果。
- `目标变化`：材料显示该方向已被用户主动降级或替换。
- `证据不足`：无法判断，需下周期补记录。

如果方向锚点是“考公 / 行测 / 申论 / 作息稳定”等长期主线，而本周期主主题没有覆盖它，必须在报告中显式说明它的状态，不能因为主题归并或轻量化而自然消失。

## 主题综合原则

1. 六问是稳定导航，不得被其他一级标题替代。
2. 主题综合只能发生在六问内部，不能反过来取代六问。
3. 材料按主题组织，不按视角顺序展开。
4. 每个重要结论都要落到具体日期、原文或视角证据。
5. 每个主题都要写出反例、限制条件或“证据不足”。
6. 复盘不只检查发生了什么，也检查重要但缺席了什么。
7. 如果没有显著矛盾或证据不足，如实记录，不强行制造。

用户明确要求“依据第一性原理分析”、复核或压缩既有复盘时，读取 `.claude/shared/contracts/first-principles-analysis.md`。复核结果必须归入聊天摘要或复盘六问的对应章节，不新增一级标题。

## 周报深度

1. 周报是“小月报”：仍保留复盘六问，但只聚焦 `1-3` 个重点。
2. 原因分析只做 3Why，不深挖到月度级信念层。
3. `## 六、下周规划` 至少包含“目标 + 手段 + 检查方式”。
4. 周报优先把本周日反馈中的行动验证、证伪和连续没做项沉淀为下周最小实验。

## 月报深度

1. 月报在正式写报告前先归并 `2-3` 个主主题。
2. 每个保留主题补齐主题名、支持视角、关键证据、反例或证据不足说明、对重来演练或下月规划的意义。
3. `## 六、下月规划` 必须包含“目标 + 手段 + 检查点 + 假说”。
4. 如果本月材料显示长期方向冲突、重复卡点、工作观 / 人生观冲突，或下月规划只能做局部修补，必须在 `## 六、下月规划` 中增加一条基于证据的升级提醒：

```text
⚠️ 这可能不是普通执行问题：[用一句话点出证据]。建议输入 /life-design --quick 做一次人生设计校准。
```

不要自动生成 life-design 报告。
