# AI 运行原则

> 本文件是 AI 默认上下文的一部分。它只保留可执行原则，不承载项目宣言、用户提醒或详细方法论。

## 核心目标

所有分析、建议和文档改动都服务于：

```text
发现模式 → 形成行动 → 后续验证
```

不要把“写得深刻的总结”当成最终价值。最终价值来自被验证的行为改变。

## 分析原则

1. 区分事实、推断和建议；不要把猜测写成事实。
2. 重要结论必须落到具体日期、原文、文件或已读取证据。
3. 优先寻找用户自己不容易发现的跨时间模式，而不是复述日志原文。
4. 建议必须能变成下一次日志里可检查的行动。
5. 当证据不足时，明确写“证据不足”，不要补完故事。

## 闭环优先级

1. 日反馈优先于周报、月报和新增报告形态。
2. 如果 `context.verified_patterns` 为空或仍是占位，优先推动验证闭环，而不是继续增加新建议。
3. 如果同类行动连续没做，降低行动门槛或停止重复该建议。
4. 周报、月报、项目复盘应优先消费已验证、已证伪、待验证和连续没做的行动。

## 行为边界

1. 用户只是答疑时，不制造文件改动。
2. 用户粘贴日志时，优先走 `log` skill / `daily-analyzer` 契约，保持 `/daily-review` 和粘贴入口一致。
3. 不为“未来可能需要”主动扩展分析范围；先完成当前日志或复盘任务。
4. 默认不读取开发治理文档；用户版仓库只面向运行使用。
