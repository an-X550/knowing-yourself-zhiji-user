# 周期证据包处理器

按指定视角（治疗师、教练、编年史等）处理周期内的日志条目，为综合引擎产出证据包。

## 输入格式

用户提供：`"Process [MONTH] as [PERSPECTIVE]"`、`"Process week [YYYY-Www] as [PERSPECTIVE]"` 或类似变体，如：
- "以治疗师视角处理2026-04"
- "以教练视角处理 week 2026-W27"
- "分析2025全年的教练视角，逐月处理"
- "为2024年12月创建说书人材料"

## 执行步骤

### 0. 加载路径约定
读取 `.claude/shared/paths.md` 获取所有输入/输出路径约定。后续步骤中的文件路径均从此文件获取，不再硬编码。

### 1. 解析输入

从用户输入中提取：
- **周期**：优先解析 `week YYYY-Www`；否则解析月份并转换为 `YYYY-MM` 格式（如 "2025年1月" → "2025-01"）
- **视角**：要使用的角色/视角（如 "therapist"、"coach"、"storyteller"）

### 2. 加载视角定义

从以下路径读取视角定义：
```
perspectives/[perspective].md
```

如果视角文件不存在，列出该文件夹中的可用视角告知用户。

### 3. 收集日志文件

如果输入是月份：
- 找到目标月份的所有日志文件：glob `日志/*[YYYY]*[M]月*.md` 或 `日志/*[YYYY]-[MM]*.md`。

如果输入是周：
- 只收集该 ISO 周对应日期范围内的日志条目；不要因为日志文件属于同一个月就自动扩展为整月。

如果未找到，报告错误。按时间顺序排序（第一天 → 最后一天）。

### 4. 处理日志并提取证据包

**关键规则：**
- 按时间顺序处理条目（第1天 → 最后一天）
- 分析前完整阅读每篇日志
- 应用视角的问题/框架到每个条目
- 追踪当前周期内的模式、主题和演变
- 如果输入是月份，你始终只处理一个日历月；如果输入是周，你始终只处理这 7 天日期范围。不要把周度输入扩展为整月。
- 你的职责是为综合引擎准备可交叉验证的证据包，不是替它提前写最终复盘
- 只保留当前视角真正看得见的信号；弱证据要降权，单次偶发信号不要硬扩展成整月结论
- 每个关键观察都要附日期证据；如果证据不足或存在明显反例，要显式写出

### 5. 生成输出

如果输入是月份：
- 创建输出文件（路径见 `.claude/shared/paths.md` 中的"视角分析中间产物"。命名格式：`{视角}/YYYY-MM-{视角}.md`）。

如果输入是周：
- 不创建月度中间文件；直接把本周证据包作为返回文本交给上层 workflow。

**输出结构**遵循视角文件的 `# 输出格式` 段，但默认把它理解为“中间证据包格式”，不是最终报告格式。

**始终包含：**
- 带日期、视角和所处理周期的前言
- 按视角定义的清晰段落或表格
- 引用示例时标注具体日志日期（引用保留原始语言，不翻译！）
- 候选发现与对应证据
- 反例、证据不足或置信度说明（如视角模板要求）

**禁止输出：**
- 最终总结、总论、总标题式结案陈词
- 下月计划、行动建议、治疗建议、人生建议
- 跨月份对比
- 把整份材料写成可以直接交付给用户的完整月报

## 输出前言模板

```yaml
---
created: [当前日期]
perspective: [视角名称]
period: [YYYY-MM 或 YYYY-Www]
journals_processed: [计数]
---
```

## 中间产物定位

把你写的文件当作“综合材料”而不是“独立报告”：

- 目标：让后续综合引擎能按主题归并多个视角的证据
- 允许：候选模式、证据表、信号强弱、反例、证据缺口
- 不允许：代替综合引擎完成跨视角总判断

## 错误处理

- 如果周期没有日志：报告"未找到 [周期] 的日志条目"
- 如果视角未找到：列出 `perspectives/` 中的可用视角
- 如果周期格式不明确：请求澄清

**重要**：不要把创建的证据包内容读回给主代理。月度输入负责创建文件并返回完成；周度输入直接返回证据包文本供综合层使用。
