# 日志教练命令

评估指定范围内的日志写作质量，产出评分反馈和改进建议。

## 输入

日期范围来自：`$ARGUMENTS`

**默认值**：如果未提供参数，直接分析最近7天（含今天，无交互提示）。

**支持的格式**：
- `YYYY-MM-DD..YYYY-MM-DD`（如 "2026-06-28..2026-07-05"）
- `last N days`（如 "最近7天"、"last 7 days"）
- `本周`——本周一至今天
- `--ask`——交互式提示日期范围

## 执行步骤

### 1. 确定日期范围

解析参数获取要分析的日期列表。
如无参数：最近7天（今天及往前6天）。

### 2. 验证日志存在

检查范围内每个日期的日志文件。
跳过没有日志文件的日期（不报错）。
至少需要3个条目才能做有意义分析。

按顺序检查路径：
1. `日志/YYYY-MM-DD.md`（单独文件）
2. 合并文件（`*月日志.md`、`*日志.md`）

### 3. 启动质量分析

使用 Task 工具，`subagent_type: journal-quality-coach`：
```
"Coach journals [日期范围]"
```

### 4. 汇总结果

由 `journal-quality-coach` 统一完成逐篇评分、趋势汇总和报告写入。

```
日志教练报告：[日期范围]

分析条目数：N
平均得分：X/30（等级：A-F）
得分趋势：上升 / 稳定 / 下降

最强条目：[日期]（X/30）
最弱条目：[日期]（X/30）

进步最大的步骤：[步骤名]
最被忽视的步骤：[步骤名]

重复错误：
- [错误名称]：出现 N 次
- [错误名称]：出现 N 次

需要改进的前3项：
1. [具体、可行动]
2. [具体、可行动]
3. [具体、可行动]
```

### 5. 检查人生设计轻提醒

汇总最近 7 天日志质量时，额外检查是否出现高密度方向性信号：

- 反复写到“要不要继续”“这条路是不是错了”“不知道想要什么”“没有意义”
- 同一行动建议连续没做，或做了也没有改善
- 明显目标-能量冲突：追求的目标持续消耗、回避、拖延、恢复困难
- 用户把长期失衡合理化为“没办法”

如果仅是单日情绪低落或普通任务压力，不提醒。

如果命中 ≥2 条信号，在报告末尾追加：

```text
⚠️ 这可能不只是日志质量问题。最近 7 天出现方向性卡点信号，建议输入 /life-design --quick 做一次轻量校准。
```

### 6. 写入报告

保存汇总报告到：
```
复盘/每日反馈/coach-report-YYYY-MM-DD.md
```

## 错误处理

- 日志少于3篇："范围内仅找到 N 篇日志条目。至少需要3篇才能做有意义的教练分析。"
- 完全没有日志："在 [范围] 内未找到日志条目。请先写一些日志！"
