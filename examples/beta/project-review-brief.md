# 项目复盘样例：用户版内测验收

## 项目背景

我们把开发者自用的“知己”项目裁剪成一个用户版仓库，目标是发给 3-5 位内测用户试用。

## 本轮目标

1. 不暴露开发治理文档和个人真实数据
2. 保留用户真正会用到的功能入口
3. 让用户按 README 就能开始使用
4. 至少覆盖日分析、周复盘、月复盘、项目复盘、人生设计这些主入口

## 已完成

- 删除了大部分开发态说明
- README 改成面向用户，并覆盖用户首次使用所需的必要说明
- 补齐了周/月复盘依赖的 perspectives
- 补齐了 关于我/ 模板文件
- 增加了日志教练链路

## 暴露过的问题

- 从上级目录复制时，部分运行依赖没有一起带下来
- 有些命令还保留开发态描述或旧路径
- 有些文档说法和命令默认行为不一致
- 周/月入口缺少给内测用户直接试跑的样例材料

## 这次验收最在意什么

- 用户能不能自己开始用
- 会不会点开命令就因为缺文件或错路径失败
- 哪些问题会阻塞 3-5 人内测
- 哪些地方还会让用户误以为自己拿到的是开发版
