# 视角体系

此目录包含两种架构不同的视角文件，它们在月度复盘流水线中扮演不同角色。

## 提取器视角（6个）— 数据采集层

`chronicle` / `coach` / `therapist` / `relationships` / `strengths` / `values-meaning`

- **职责**：从日志中提取结构化数据，供综合引擎（synthesis）消费
- **共同约束**：不跨期对比、不做最终判断、不写最终总结、不抢最终月报叙事
- **输出类型**：结构化数据表 + 候选发现 + 证据边界说明
- **架构**：角色 → 数据提取指令 → 初步判断指令（含禁止项） → 输出模板 → 语气 → 规则

## 评估器视角（3个）— 独立评估层

`growth-dimensions` / `journal-quality` / `review-coach`

- **职责**：按方法论标准独立评分（30分制），产出评分 + 诊断
- **共同约束**：评分必须有文本证据、不分析日志生活内容本身
- **输出类型**：评分量表 + 错误检测 + 改进建议
- **架构**：角色 → 评分框架 → 评分量表 → 常见错误检测 → 输出模板 → 语气 → 规则

## 共同规则

以下规则适用于所有 9 个视角文件：
- 所有输出使用简体中文（引用日志原文除外）
- 每个关键论断必须有日期引用
- 不与其他时期对比（跨时期对比是综合引擎的职责）
- 使用原文引用，不翻译日志内容
